# Desafios de Visualização de Dados

**5º Período — UNIFSA**  
**Disciplina: Análise Avançada de Dados**

## Descrição da Atividade

Esta atividade tem como foco a aplicação prática de conceitos fundamentais de **visualização de dados** com Python, utilizando principalmente as bibliotecas **Matplotlib** e **Seaborn**.

Ao longo dos desafios, foram construídos e refatorados diferentes tipos de gráficos com o objetivo de representar informações de forma mais **clara**, **honesta** e **eficiente**. Além da implementação dos gráficos, a atividade também exige uma análise crítica baseada em princípios teóricos de visualização.

Entre os referenciais utilizados, destacam-se:

- **Cleveland-McGill**, com ênfase na eficácia das codificações visuais;
- **Gestalt**, especialmente nos princípios de percepção visual;
- **Edward Tufte**, com conceitos como **Data-Ink Ratio**, **Lie Factor** e **Chartjunk**.

## Objetivo

Aplicar boas práticas de visualização de dados para selecionar, construir e refatorar gráficos de acordo com o tipo de informação representada, buscando melhorar a interpretação visual e evitar distorções.

A análise desenvolvida ao longo da atividade busca responder, principalmente, às seguintes questões:

- Qual tipo de gráfico é mais adequado para cada cenário?
- Como representar os dados de forma mais clara e honesta?
- Quais problemas visuais podem prejudicar a interpretação do gráfico?
- Como princípios teóricos de visualização ajudam a justificar escolhas de design?

## Estrutura da Atividade

A atividade foi dividida em duas partes principais:

### Parte 1 — Escolha do Gráfico Adequado
Nesta etapa, o foco foi selecionar o tipo de gráfico mais apropriado para diferentes conjuntos de dados e contextos analíticos.

Foram construídas visualizações para representar:

- Comparações entre categorias;
- Distribuição de valores;
- Tendências ao longo do tempo;
- Relações entre variáveis;
- Proporções e composições.

Além da implementação, cada escolha foi justificada com base em princípios de percepção visual e eficácia comunicacional.

### Parte 2 — Refatoração Crítica de Gráficos
Nesta etapa, foram analisados gráficos propositalmente problemáticos, com o objetivo de identificar falhas de design e propor versões refatoradas mais adequadas.

Entre os principais problemas avaliados, estão:

- Uso inadequado de escalas e eixos;
- Excesso de elementos visuais desnecessários;
- Cores sem significado analítico;
- Poluição visual;
- Baixa legibilidade;
- Escolhas de gráficos pouco eficazes para o tipo de comparação proposta.

Cada refatoração foi acompanhada de justificativas baseadas em teoria, buscando tornar os gráficos mais eficientes, informativos e visualmente claros.

## Contexto da Análise

No papel de analista de dados, a proposta da atividade é desenvolver não apenas a habilidade técnica de gerar gráficos, mas também a capacidade crítica de avaliar a qualidade de uma visualização.

Mais do que “fazer gráficos bonitos”, o objetivo é compreender como escolhas de design influenciam diretamente a interpretação dos dados e a tomada de decisão.

Ao final, espera-se que a atividade contribua para uma visão mais analítica sobre visualização de dados, unindo **programação**, **percepção visual** e **comunicação de informação**.
